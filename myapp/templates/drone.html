AI Drone Tag · Interactive brief

Teaching a drone to catch a moving target

A flight pilot that's learned, not programmed — trained entirely in simulation, with a control interface built to fly real hardware.

39-D observation 4-D action 100 Hz control 256×256 policy PPO model-free

Catching a drone that doesn't want to be caught is a control problem that resists hand-written rules. A target that weaves, brakes, and breaks sideways the moment it senses a threat will beat any fixed "if it goes left, follow left" logic. AI Drone Tag takes the other route: the interceptor's pilot is a neural network that learns to chase, discovered through reward and repetition rather than written by hand. It trains entirely in simulation — millions of chases — and its output is the same command a real flight controller already speaks, so the learned pilot is built from the start to leave the sim and fly real hardware.

01 The system

One loop, run a million times

Every bit of behaviour comes out of a single closed loop. The drone senses its situation as a vector of numbers, a compact network turns that into a flight command, the simulated chase plays out the physics against an evading target, and a reward signal quietly tunes the network toward catching it. Run that loop enough and a pilot emerges — no chase logic was ever written, only a definition of what "good" looks like.

FIG 01 — sense → decide → act → reward, closed.

02 The curriculum

Learning to fly, then to hunt

You can't put a fresh network in front of a tactical, full-speed evader and expect it to learn — it would fail every episode and pick up nothing. So the task is split into three blocks of rising difficulty. Tap a block to see its real training phases.

03 Earning progression

Mastery, not a schedule

Progress through the curriculum is earned. Before reinforcement learning even begins, the pilot is warm-started by imitating a simple scripted expert — but only on flying, never on the chase, so the hunting is always learned. A mastery controller then holds the agent in each stage and advances it only once its rolling success rate clears the stage's gate. Hover a band for what it trains.

FIG 03 — every stage is a gate; the agent only advances by clearing it.

04 Inside one step

A hundredth of a second

Underneath, one tick of the pilot is small and fast: read the true state, pack it into a 39-number vector, run a compact two-layer network, and read out four numbers that decode into collective thrust and body rates — the low-level language of any flight controller. Run a tick to watch it flow.

39 in → 4 out, at 100 Hz

05 Why it matters

Built to leave the simulation

The detail that turns a simulation into a path to real autonomy is the shape of those four output numbers. Collective thrust plus body rates is exactly the interface a real drone's inner control loop already accepts — so moving from sim to hardware doesn't mean redesigning the pilot, only swapping the physics underneath it for the real world.

That makes this a software-first autonomy layer: the intelligence that decides how to fly, sitting on top of hardware that already knows how to execute — not another piece of counter-drone hardware. The same learned pilot that catches an evader in simulation is the one you'd carry onboard.

From a definition of "good" to a pilot that hunts — learned in simulation, built to fly.
